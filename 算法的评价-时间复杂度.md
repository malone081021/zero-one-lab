## 方法

事前评价算法的好坏

执行的效率，随着数据量的增加，算法执行的时间的变化趋势

如果随着数据规模的增加，算法执行的时间急剧增加，那就不是一个好的算法；

这种特性可以使用函数的斜率表示，所以算法的复杂性都是使用函数表示的；

$n^2$ ， n*$\ln n$，n, $\ln n$



---

根据算法的表达，估计算法最主要部分执行的次数和数据规模之间的关系



``` java 
int cal(int n) {
   int sum = 0;
   int i = 1;
   for (; i <= n; ++i) {
     sum = sum + i; // 这行代码至少执行n次
   }
   return sum;
}
```

所以这个代码或者算法的时间复杂度的表示就是n
